## 0210. Upgrade cc-skills to rd3:cc-skills

### Background

After several months, the open standards for agent skills already iterated to another level, we need to upgrade our cc-skills to support these new standards. These developments include but not limited to:
- [Agent Skills Open Standard](https://agentskills.io/home) - Now adopted by 30+ AI coding agents
- [Claude Code Skill Creator 2.0](https://github.com/anthropics/skills/tree/main/skills/skill-creator)
- [Codex Skill Creator](https://github.com/openai/codex/tree/main/codex-rs/skills/src/assets/samples/skill-creator)

Based on these changes, I already had some research on it. You can refer to the research report at @docs/reasearch/how_to_write_skills_en.md and docs/reasearch/on_skill_creation.md.

Meanwhile, I downloaded the source code of two versions of the skill creator here:
- Claude Code Skill Creator 2.0: vendors/skills/skills/skill-creator
- Codex Skill Creator: vendors/codex/codex-rs/skills/src/assets/samples/skill-creator

We already have our cc-skills at @plugins/rd2/skills/cc-skills. It works well with the claude code, but we also have several issues or limitations with it, including:
- Claude Code only. We created cc-skills before Agent Skill becomes an open standard. More and more coding agents are built on top of Agent Skill, and we need to support them.
- The Pre-Skill Creator. Current cc-skills is built before these official skill creators.
- Current rules for validation are too strict. That will cause no extendability.
- The evaluation dimension and scoring are not logically consistent. That cause more and more adjustment errors to refine the new skills despite the new skills are generated by the skill creator itself.
- plugin rd2 involve so many programming code with bash, python and typescript. We need to reduce the complexity and make it more extensible. That's reason we created a new plugin rd3, which will replace rd2 by removing python scripts and adding more coding agents support and etc.

We need to take this chance to mitigate these issues and upgrade it to support the new open standards.

### Requirements

Based on these two official skill creators and existing rd2:cc-skills(plugins/rd2/skills/cc-skills) to create a new Agent skill rd3:cc-skills at plugins/rd3/skills/cc-skills.

#### Core Requirements
- Consolidate both Claude Code Skill Creator 2.0 and Codex Skill Creator as the main frame for the new skill creator
- Refer to rd2:cc-skills for the following features:
  - Template based skill creation
  - The framework of evaluation, but the dimension and scoring logic should be refined based on these official skill creators
  - Scripting support for evaluation and scoring
- Support multiple coding agents: Claude Code, Codex, Antigravity, OpenCode, OpenClaw (pi-mono compatible)
- Tech stack: Bun.js + TypeScript + Biome (no Python)
- All common scripts in `plugins/rd3/scripts`, skill scripts in `plugins/rd3/skills/<skill_name>/scripts`

#### Key Design Goals
1. **Universal Skill Creator** - Create skills that work across ALL platforms from a single source of truth
2. **Single Source of Truth** - ONE SKILL.md file, platform companions are additive (not alternative versions)
3. **Backward Compatible** - Can refine/enhance existing rd2 skills with migration support
4. **Extensible Plugin System** - Platform adapters can be added independently

### Q&A

**Q: How do platform adapters maintain single source of truth?**
A: Platform adapters generate *companion files* (like `agents/openai.yaml`) that sit alongside SKILL.md. They never create alternative versions of the skill content. The base SKILL.md with `name` and `description` works on all 30+ platforms.

**Q: Can rd3:cc-skills refine existing rd2 skills?**
A: Yes. The `--migrate` flag detects rd2-specific patterns and adds multi-platform support while preserving all existing content and Claude Code features.

**Q: What happens to Claude Code-specific syntax like `!`cmd`` and `$ARGUMENTS`?**
A: They are preserved for Claude Code and documented as Claude-only in a Platform Notes section. Other platforms execute equivalent commands via Bash tool.

#### Migration Support (rd2 → rd3)

| rd2 Feature | Detection | Migration Action |
|-------------|-----------|------------------|
| `!`cmd`` syntax | Regex scan | Keep for Claude, add Platform Notes |
| `$ARGUMENTS`, `$N` | Regex scan | Keep for Claude, document limitation |
| Missing `name:` field | Frontmatter check | Add explicit `name:` from directory |
| `context: fork` | Frontmatter check | Keep (ignored by others) |
| `hooks:` field | Frontmatter check | Keep or move to comment |
| Python scripts | File extension | Keep (scripts are platform-agnostic) |

```bash
# Migrate existing rd2 skill
/rd3:skill-refine skills/my-skill --migrate --platform all
# → Adds explicit `name:` if missing
# → Adds `metadata.openclaw` for OpenClaw
# → Generates `agents/openai.yaml` for Codex
# → Preserves all existing content
# → Documents Claude-specific features
```

#### Command Reference

```bash
# Create new skill
/rd3:skill-add my-skill --template technique

# Evaluate skill (basic = validate only, full = test cases + eval)
/rd3:skill-evaluate my-skill --scope basic
/rd3:skill-evaluate my-skill --scope full --platform all

# Refine skill
/rd3:skill-refine my-skill --from-eval eval-results.json
/rd3:skill-refine my-skill --migrate --platform codex,openclaw

# Package for distribution
/rd3:skill-package my-skill --platform all --output ./dist/
```

#### Evaluation Framework

**Scope: basic** (default)
- Frontmatter validation (name, description, metadata)
- Structure validation (scripts/, references/, assets/)
- Best practices check (line count, progressive disclosure)
- Platform compatibility check

**Scope: full**
- All basic checks +
- Test case execution (via subagents)
- Quality scoring across dimensions
- Eval report generation
