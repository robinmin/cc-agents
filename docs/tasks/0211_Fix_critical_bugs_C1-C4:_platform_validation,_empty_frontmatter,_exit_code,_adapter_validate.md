## Batch 1: Critical Bug Fixes (C1-C4)

These 4 issues produce incorrect behavior and must be fixed first.

### C1: Platform validation never runs (evaluate.ts)
- `_evaluatePlatform` (line 903) is defined but never called
- The `_platforms` parameter in the main evaluation flow is unused
- **Fix**: Wire `_evaluatePlatform` into the evaluation pipeline, call it when `--platform` flag is provided
- **Files**: `scripts/evaluate.ts`

### C2: Empty frontmatter passed to adapters (refine.ts)
- Line 181: `frontmatter: {} as SkillFrontmatter` passes empty object
- **Fix**: Parse actual frontmatter from skill file and pass it to adapters
- **Files**: `scripts/refine.ts`

### C3: Rejected skills exit with code 0 (evaluate.ts)
- Line 1248: `report.passed || report.rejected ? 0 : 1` — rejected exits success
- **Fix**: Change to `report.passed ? 0 : 1` so rejected skills exit with code 1
- **Files**: `scripts/evaluate.ts`

### C4: BaseAdapter.validate checks filename not directory (base.ts)
- Line 37-38: `skill.path.split('/').pop()` returns `SKILL.md`, not directory name
- **Fix**: Use `skill.directory` or extract directory name from path
- **Files**: `scripts/adapters/base.ts`
