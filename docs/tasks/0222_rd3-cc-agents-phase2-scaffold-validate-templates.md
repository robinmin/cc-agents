## 0222. rd3-cc-agents-phase2-scaffold-validate-templates

### Background

Phase 2 of rd3:cc-agents skill creation. Implements the scaffold (agent generation from templates), validate (agent quality checking), and 3 tiered templates (minimal/standard/specialist). This replaces the old rd2:cc-agents mandatory 8-section anatomy with flexible tiered templates. Templates define the structure skeleton for each tier; scaffold fills them with user-provided content; validate checks structural and content quality. Must follow rd3:cc-skills scaffold/validate patterns. Parent task: 0220.

### Requirements

Implement templates/ directory with 3 agent templates: minimal.ts (name, description, basic rules - for simple task agents), standard.ts (adds persona, competencies, process - for general-purpose agents), specialist.ts (full 8-section anatomy with verification protocol - for domain expert agents). Implement scaffold.ts that takes template tier + user inputs and generates a complete agent definition in UAM format. Implement validate.ts that checks agent definitions against structural rules, content quality heuristics (description clarity, competency count, rule coverage), and tier-specific requirements. Add test fixtures with sample agents for each tier and each platform format. Success criteria: scaffold generates valid agents for all 3 tiers, validate catches common issues (empty sections, missing required fields, weak descriptions), test fixtures pass validation.
