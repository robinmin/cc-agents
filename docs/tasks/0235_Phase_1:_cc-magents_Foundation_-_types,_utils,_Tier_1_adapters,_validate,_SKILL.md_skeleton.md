### Design


Follow rd3:cc-agents architecture patterns. UMAM is section-based (unlike UAM which is frontmatter-based). Adapter interface supports tiered platforms. Validation is structural only (quality scoring is in evaluate.ts Phase 2).

### Solution


Create the foundation layer for rd3:cc-magents at plugins/rd3/skills/cc-magents/:

**Files to create:**
1. `scripts/types.ts` - UMAM types: UniversalMainAgent, MagentSection, SectionCategory (14 types), MagentPlatform (23+ platforms with tiered support), adapter interfaces (IMagentPlatformAdapter with parse/validate/generate/detectFeatures/getDiscoveryPaths), evaluation types (scores, grades, dimensions), evolution types (proposals, safety levels)
2. `scripts/utils.ts` - Markdown parsing (split sections by headings), section detection/extraction, section serialization back to markdown, heading level normalization, token estimation (word count * 1.3), file discovery (scan for agent configs), platform auto-detection from file path/content
3. `scripts/adapters/base.ts` - Abstract BaseMagentAdapter implementing IMagentPlatformAdapter with common validation logic
4. `scripts/adapters/agents-md.ts` - Tier 1 canonical AGENTS.md adapter (parse any valid markdown per AGENTS.md spec, generate standards-compliant output, detect AGENTS.md-specific features like metadata comments)
5. `scripts/adapters/claude-md.ts` - Tier 1 CLAUDE.md adapter (parse Claude-specific sections like hooks/skills/memory, generate with Claude conventions)
6. `scripts/adapters/index.ts` - Adapter registry mapping MagentPlatform to adapter instances
7. `scripts/validate.ts` - Structural validation: markdown parseable, file size within limits, no embedded secrets (regex patterns for API keys, tokens), section structure analysis, platform-specific structural rules
8. `SKILL.md` - Skeleton with metadata header, description, trigger phrases, placeholder operation sections

**Architecture patterns (from rd3:cc-agents reference):**
- TypeScript with Bun runtime
- No external deps beyond yaml and marked (already in monorepo)
- Export clean interfaces for downstream scripts
- Adapter pattern with registry for extensibility
- Validation returns structured results (errors[], warnings[], messages[])

### Plan


1. Create directory structure plugins/rd3/skills/cc-magents/scripts/adapters/
2. Implement types.ts first (all other files depend on it)
3. Implement utils.ts (parsing/serialization foundation)
4. Implement adapters/base.ts, then agents-md.ts and claude-md.ts, then index.ts
5. Implement validate.ts using utils and adapters
6. Create SKILL.md skeleton
7. Run biome lint and tsc --noEmit to verify
