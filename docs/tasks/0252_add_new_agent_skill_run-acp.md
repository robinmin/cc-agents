## 0252. add new agent skill run-acp

### Background

This task tracks the rd3 `run-acp` skill, which wraps the vendored `acpx` project under `vendors/acpx` for agent-to-agent delegation over ACP.

The original scaffold existed, but review uncovered several documentation defects:

- Invalid `--agent` examples
- Incorrect `exec` equivalence guidance
- Incorrect TTL semantics
- Incorrect session-resume guidance
- Missing coverage for real CLI features already supported by `acpx`

This cleanup keeps the rd3 skill documentation aligned with the actual CLI so downstream rd3 skills can safely reference `run-acp` as a reusable tool-wrapper.

### Requirements

- Keep `plugins/rd3/skills/run-acp/` aligned with the vendored `acpx` source and live CLI help.
- Ensure examples are executable and do not document invalid command combinations.
- Document the supported command surface that rd3 users will rely on: prompt, exec, status, session lifecycle, queueing, permissions, output modes, config keys, and custom adapters.
- Update the task record so its sections, artifacts, and status reflect the implemented skill and this review-driven correction pass.


### Q&A



### Design

The rd3 skill is a documentation-first wrapper around the vendored `acpx` CLI rather than a new runtime implementation.

Design decisions:

- Use `vendors/acpx` plus live `acpx --help` as the source of truth for command grammar and flags.
- Keep the rd3 skill focused on accurate usage guidance rather than re-documenting unsupported behavior.
- Preserve rd3 frontmatter and platform metadata while aligning the body content with real `acpx` semantics.



### Solution

Implemented `plugins/rd3/skills/run-acp/` as a source-aligned `acpx` tool-wrapper skill and corrected the issues found during review.

Review-driven fixes applied in this pass:

- Replaced drifted command guidance in `SKILL.md` with a CLI-accurate command model.
- Restored missing documented capabilities such as `sessions ensure`, `sessions read`, `--resume-session`, `--auth-policy`, `--non-interactive-permissions`, `--model`, `--allowed-tools`, `--max-turns`, and `disableExec` config notes.
- Corrected invalid examples involving `--agent` plus positional agents.
- Corrected session TTL semantics and session resume guidance.
- Synced the built-in registry docs with the actual `acpx` registry, including `iflow` and droid aliases.
