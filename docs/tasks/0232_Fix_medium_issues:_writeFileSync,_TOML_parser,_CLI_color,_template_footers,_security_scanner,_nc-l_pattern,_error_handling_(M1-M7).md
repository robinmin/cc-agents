## 0232. Fix medium issues: writeFileSync, TOML parser, CLI color, template footers, security scanner, nc-l pattern, error handling (M1-M7)

### Background

Code review found 7 MEDIUM severity issues across multiple files. refine.ts uses writeFileSync instead of Bun.file() (M1). Codex TOML parser can't handle multi-line strings/arrays/nested tables (M2). scaffold.ts --color flag parsed but not passed (M3). Templates have trailing metadata footers (M4). Security scanner regex too broad causing false positives (M5). evaluation.config.ts nc -l pattern slightly differs from spec (M6). utils.ts readAgent() lacks error handling for parse failures (M7).

### Requirements

M1: Replace writeFileSync in refine.ts with Bun.file().write() for consistent I/O. M2: Improve Codex TOML parser to handle multi-line strings, arrays, and nested tables (or use a proper TOML library). M3: Wire scaffold.ts --color CLI flag through to the scaffold function so it actually affects output. M4: Remove trailing metadata/footer lines from templates that would appear in scaffolded agents. M5: Narrow security scanner regex pattern to reduce false positives on agent docs discussing passwords/secrets conceptually. M6: Fix evaluation.config.ts nc -l pattern to match spec exactly. M7: Add error handling in utils.ts readAgent() for frontmatter parse failures (malformed YAML, missing delimiters).
