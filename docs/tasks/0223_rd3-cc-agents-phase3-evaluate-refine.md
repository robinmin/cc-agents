## 0223. rd3-cc-agents-phase3-evaluate-refine

### Background

Phase 3 of rd3:cc-agents skill creation. Implements the evaluation system (8 dimensions with 2 weight profiles) and the refine pipeline (automated improvement suggestions). The evaluation system scores agent definitions across quality dimensions; refine takes evaluation results and generates specific improvement actions. This follows the same evaluate/refine pattern as rd3:cc-skills but with agent-specific dimensions. Parent task: 0220.

### Requirements

Implement evaluation.config.ts defining 8 evaluation dimensions for agents: (1) Trigger Design - description clarity and routing accuracy, (2) Instruction Clarity - persona/process/rules specificity, (3) Competency Coverage - breadth and depth of listed competencies, (4) Verification Protocol - quality gates and red flags, (5) Tool Restriction Compliance - tools whitelist/blacklist correctness, (6) Delegation Pattern - proper skill invocation vs direct implementation, (7) Output Format - structured output templates and examples, (8) Cross-Platform Portability - UAM completeness for multi-platform generation. Define 2 weight profiles: thin-wrapper (emphasizes delegation, trigger design) and specialist (emphasizes competencies, verification, instruction clarity). Implement evaluate.ts that scores an agent definition across all 8 dimensions with per-dimension scores and weighted total. Implement refine.ts that takes evaluation results and generates specific improvement suggestions with before/after examples. Success criteria: evaluate produces meaningful differentiation between good and poor agents, refine suggestions are actionable and specific.
