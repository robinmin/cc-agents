## 0225. rd3-cc-agents-phase5-skill-md-references-integration

### Background

Phase 5 of rd3:cc-agents skill creation. Creates the SKILL.md definition file, references/ documentation directory, sample platform-specific agent files (agents/openai.yaml, metadata.openclaw), and integrates everything into a cohesive skill. This is the final assembly phase that produces the user-facing skill documentation and ensures all components work together. Parent task: 0220.

### Requirements

Create SKILL.md following rd3:cc-skills SKILL.md pattern: frontmatter with name/version/description/triggers, Overview, Quick Start, Pipeline Architecture (scaffold -> validate -> evaluate -> refine -> adapt), Platform Support Matrix, Template Tiers, Evaluation Dimensions, Usage Examples, Related Skills. Create references/ directory with: architecture.md (system design), templates.md (tier details), evaluation.md (dimension details), adapters.md (platform format specs), examples.md (usage examples). Create sample output files: agents/openai.yaml (OpenAI-compatible agent definition), metadata.openclaw (OpenClaw agent metadata). Verify end-to-end pipeline: scaffold a new agent -> validate it -> evaluate it -> refine it -> adapt to all 6 platforms. Success criteria: SKILL.md is under 3000 tokens, references provide comprehensive detail, sample files are valid for their platforms, end-to-end pipeline completes without errors.
