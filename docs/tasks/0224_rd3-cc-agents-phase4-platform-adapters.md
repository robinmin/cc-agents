## 0224. rd3-cc-agents-phase4-platform-adapters

### Background

Phase 4 of rd3:cc-agents skill creation. Implements the remaining 5 platform adapters (Gemini, OpenCode, Codex, OpenClaw, Antigravity) and the adapt.ts cross-platform conversion pipeline. Each adapter provides bidirectional conversion between the platform's native agent format and the UAM. The adapt pipeline enables converting agents between any two platforms via UAM as intermediate. Parent task: 0220.

### Requirements

Implement adapters/gemini.ts for Gemini agent format (system instructions YAML/JSON with persona, capabilities, constraints). Implement adapters/opencode.ts for OpenCode agents.md format (frontmatter + markdown body). Implement adapters/codex.ts for Codex agent configuration format (AGENTS.md with role definitions). Implement adapters/openclaw.ts for OpenClaw metadata format (YAML-based agent definitions with capabilities and restrictions). Implement adapters/antigravity.ts for Antigravity agent format (structured agent cards with skills and workflows). Implement adapt.ts that orchestrates source_adapter.parse() -> UAM -> target_adapter.generate() with loss detection (fields that cannot be represented in target format). Implement adapters/index.ts barrel export. Success criteria: each adapter can round-trip its native format through UAM, adapt.ts converts between any pair of platforms, loss detection reports what fields are dropped or approximated.
