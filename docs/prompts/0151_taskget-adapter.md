## 0151. TaskGet Adapter Implementation

### Background

The TaskGet tool retrieves detailed information about a specific task by its ID. This is essential for:

1. **Task Details**: Getting full metadata for a specific task
2. **Pre-Update Validation**: Checking task exists before updating
3. **Dependency Resolution**: Fetching dependency information
4. **Debugging**: Inspecting task state for troubleshooting

The TaskGet adapter provides single-task retrieval capabilities that complement TaskList's multi-task view.

### Requirements

1. **TaskGet Tool Integration**: Properly call TaskGet with task ID
2. **Response Parsing**: Parse TaskGet response and convert to internal Task format
3. **Error Handling**: Handle task not found and tool unavailability
4. **Full Metadata Support**: Extract all available metadata fields
5. **Dependencies Extraction**: Properly parse dependency information
6. **Timestamps**: Include createdAt and updatedAt fields
