# Platform Adapters Guide

This guide explains how to create and register platform adapters for rd3:cc-skills.

## Overview

Platform adapters enable rd3:cc-skills to support multiple coding agents (Claude Code, Codex, Antigravity, OpenCode, OpenClaw) while maintaining a single source of truth.

Each adapter handles:
- **Validation**: Platform-specific skill validation rules
- **Companion Generation**: Creating platform-specific files (e.g., `agents/openai.yaml`)
- **Feature Detection**: Identifying platform-specific syntax in skills

## Adapter Interface

All adapters implement `IPlatformAdapter`:

```typescript
interface IPlatformAdapter {
  platform: Platform;           // Platform identifier
  displayName: string;          // Human-readable name

  validate(skill: Skill): Promise<ValidationResult>;
  generateCompanions(context: AdapterContext): Promise<AdapterResult>;
  detectPlatformFeatures(skill: Skill): string[];
}
```

## Creating a New Adapter

### Step 1: Create the Adapter File

Create `plugins/rd3/skills/cc-skills/adapters/<platform>.ts`:

```typescript
import { BaseAdapter, Skill, ValidationResult, AdapterContext, AdapterResult } from "./base";
import type { Platform } from "../scripts/types";

export class MyPlatformAdapter extends BaseAdapter {
  readonly platform: Platform = "my-platform";
  readonly displayName = "My Platform";

  protected async validatePlatform(skill: Skill): Promise<ValidationResult> {
    const errors: string[] = [];
    const warnings: string[] = [];

    // Add platform-specific validation rules
    // Example: Check for required fields
    if (!skill.frontmatter.metadata?.version) {
      warnings.push("My Platform recommends specifying 'metadata.version'");
    }

    return { valid: errors.length === 0, errors, warnings };
  }

  protected async generatePlatformCompanions(
    context: AdapterContext
  ): Promise<Record<string, string>> {
    const files: Record<string, string> = {};

    // Generate platform-specific companion files
    // Example: Generate a config file
    files["config.my-platform.json"] = JSON.stringify({
      name: context.skillName,
      version: context.frontmatter.metadata?.version || "1.0",
    }, null, 2);

    return files;
  }

  protected detectPlatformSpecificFeatures(skill: Skill): string[] {
    const features: string[] = [];

    // Detect platform-specific syntax
    if (skill.body.includes("$MY_PLATFORM_ARG")) {
      features.push("Uses $MY_PLATFORM_ARG substitution");
    }

    return features;
  }
}
```

### Step 2: Register the Adapter

Adapters are automatically registered when imported. In a future phase, the scaffold command will handle registration.

For now, you can register manually:

```typescript
import { adapterRegistry } from "./base";
import { MyPlatformAdapter } from "./my-platform";

adapterRegistry.register(new MyPlatformAdapter());
```

## Platform-Specific Features

### Claude Code (`claude`)
- **Extensions**: `!`cmd`` syntax, `$ARGUMENTS`, `$N`, `context: fork`, `hooks:`, `disable-model-invocation`
- **Discovery**: `.claude/skills/`
- **Companion**: None (native support)

### Codex (`codex`)
- **Extensions**: `agents/openai.yaml` (UI metadata)
- **Discovery**: `.codex/skills/`, `.agents/skills/`
- **Companion**: `agents/openai.yaml` with `display_name`, `short_description`, `default_prompt`

### Antigravity (`antigravity`)
- **Extensions**: Gemini CLI compatible
- **Discovery**: `.gemini/skills/`, `.agents/skills/`
- **Companion**: None (validates compatibility)

### OpenCode (`opencode`)
- **Extensions**: Config-level `permission.skill`
- **Discovery**: `.opencode/skills/`, `.agents/skills/`
- **Companion**: Permission hints configuration

### OpenClaw (`openclaw`)
- **Extensions**: `metadata.openclaw` JSON in frontmatter
- **Discovery**: `<workspace>/skills/`, `~/.openclaw/skills/`
- **Companion**: Extracts from `metadata.openclaw` in SKILL.md

## Validation Rules by Platform

| Platform | Required Fields | Validation |
|----------|-----------------|------------|
| All | `name`, `description` | Base validation |
| Claude Code | None additional | Warns on platform-specific syntax for other platforms |
| Codex | `name` matches directory | Rejects unknown frontmatter fields |
| Antigravity | Same as Gemini CLI | Validates compatibility |
| OpenCode | None additional | Generates permission hints |
| OpenClaw | `metadata.openclaw` optional | Extracts OpenClaw metadata |

## Companion File Patterns

### Codex: agents/openai.yaml

```yaml
# Generated by rd3:cc-skills
name: my-skill
interface:
  display_name: My Skill
  short_description: Short description for UI
  default_prompt: Default prompt when user invokes skill
```

### OpenClaw: metadata.openclaw

```yaml
metadata:
  openclaw:
    emoji: "..."
    requires:
      bins: ["node"]
```

## Testing Your Adapter

1. Create a test skill with platform-specific features
2. Run validation: `bun run scaffold.ts validate <skill-path> --platform my-platform`
3. Generate companions: `bun run scaffold.ts companions <skill-path> --platform my-platform`
4. Verify companion files are created correctly

## Best Practices

1. **Keep validation minimal**: Only enforce what the platform requires
2. **Document limitations**: Use warnings for non-critical issues
3. **Generate clean companions**: Follow platform conventions exactly
4. **Test edge cases**: Handle missing fields gracefully
5. **Preserve compatibility**: Don't break cross-platform skills
