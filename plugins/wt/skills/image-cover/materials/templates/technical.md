# Style Template: Technical

## Description

Professional, precise covers for technical articles, documentation, and architecture content. Clean aesthetic with emphasis on clarity and technical accuracy.

## Visual Characteristics

- **Colors**: Blues, grays, white background
- **Typography**: Clean sans-serif, technical font
- **Layout**: Organized, hierarchical, structured
- **Elements**: Diagrams, code snippets, technical symbols
- **Style**: Minimalist, professional, precise

## Prompt Template

```
{title}, {themes}, technical article cover, clean technical illustration, professional, precise, architectural diagram style, 2.35:1 cinematic aspect ratio, with title text overlay
```

## Style Modifiers

```
clean technical illustration, professional, precise, architectural diagram style, minimalist, organized, structured, clear visual hierarchy
```

## Text Overlay

- **Font**: Sans-serif (Roboto, Inter, system-ui)
- **Style**: Clean, uppercase, bold
- **Position**: Center or left-aligned
- **Color**: White or dark gray (high contrast)

## Color Palette

| Color | Usage |
|-------|-------|
| #007ACC | Primary blue (technical) |
| #2D2D2D | Dark gray (text) |
| #F5F5F5 | Light gray (background) |
| #FFFFFF | White (highlights) |

## Best For

- Technical documentation
- Architecture articles
- API documentation
- System design guides
- Developer tutorials

## Example

**Article**: "Microservices Architecture Patterns"

**Analysis**:
- Title: "Microservices Architecture Patterns"
- Themes: microservices, architecture, distributed systems, patterns
- Content Type: technical

**Enhanced Prompt**:
```
Microservices Architecture Patterns, technical article cover, clean technical illustration, professional, precise, architectural diagram style, showing microservices components, API gateway, services, databases, with diagram elements, 2.35:1 cinematic aspect ratio, with title text overlay: "Microservices Architecture Patterns"
```

## Detection Heuristics

Auto-detect technical style when article contains:
- Code blocks (``` or ~~~)
- Technical terms (API, architecture, patterns, system)
- Diagram mentions (architecture, topology, schema)
- Formal tone with structured content
